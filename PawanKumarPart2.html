📅 Day 10: OOP – Class & Object

Object Oriented Programming pillars

  • Class
  • Object
  • Polymorphism
  • Encapsulation
  • Inheritance
  • Data Abstraction

Class

  • Collection of variables (attributes) and methods (behavior).
  • Logical entity / blueprint – does not occupy memory.

Object

  • Instance of a class – created using new keyword.
  • Physical entity – occupies memory.
  • Multiple objects can be created for a single class.
class Employee {
    // variables
    int id;
    String name;
    // methods
    void work() {
        System.out.println("Employee works");
    }
}

// Creating object
Employee emp = new Employee();
emp.id = 101;
emp.name = "John";
emp.work();

// String object example
String s = new String("welcome");
int len = s.length();   // 7
String trimmed = s.replaceAll("\\s", "");   // removes spaces
📌 Note: String s = "welcome to java"; s.replaceAll("\\s","") → "welcometojava"

📅 Day 11: Methods vs Constructors

Method

  • Block of statements that perform a task.
  • Invoked explicitly through object.
  • Can have any name (not necessarily class name).
  • May or may not return a value – void if no return.
  • Can take parameters.
  • Used to specify logic.

Constructor

  • Name must be same as class name.
  • Never returns a value (not even void).
  • Automatically invoked at object creation.
  • Used to initialise variables.
  • Can take parameters (parameterized constructor).

Four types of methods (based on parameters & return)

  • No params, No return
  • No params, Return value
  • Takes params, No return
  • Takes params, Returns value
// Example: String length() – no params, returns int
String s = "welcome";
int l = s.length();   // 7

// substring(int, int) – takes params, returns String
String sub = s.substring(2,5);   // "lco"

Storing data into variables – 3 ways

  1. By using object reference variable
  2. By using method
  3. By using constructor

📅 Day 12: Polymorphism & Method Overloading

Polymorphism: One thing having many forms (e.g., water → vapour/ice, shape → rectangle/circle).

Method Overloading (Compile‑time Polymorphism)

Multiple methods with the same name but different parameters.

  • Method names must be same.
  • Number of parameters must be different.
  • Data types of parameters must be different.
  • Order of parameters must be different.
class X {
    void m1() { }
    void m1(int a) { }          // different number
    void m1(String s) { }       // different type
    void m1(int a, String b) { }
    void m1(String b, int a) { } // different order
}
❓ Can we overload main()? Yes, but the JVM always calls public static void main(String[] args) as entry point.

📅 Day 13: this keyword, Encapsulation, static keyword

Types of variables

  • Class variables / Instance variables – declared inside class but outside methods.
  • Local variables – declared inside methods/constructors.

this keyword

Represents the current class instance – used to refer to instance variables when shadowed by local variables.

Encapsulation

  • All variables should be private.
  • For each variable, provide getter and setter methods.
  • Variables can be operated only through methods.
class Account {
    private double balance;
    public void setBalance(double b) { if(b>0) balance = b; }
    public double getBalance() { return balance; }
}

static keyword

  • Static methods can access static stuff directly (without object).
  • Static methods can access non‑static stuff through object.
  • Non‑static methods can access everything directly.
class Test {
    static String s = "welcome";
}
// Access static member via class name
Test.s.length();

// System.out is a static PrintStream object
System.out.println("Hello");

📅 Day 14: Inheritance & Variations of main()

Inheritance: Acquiring properties (variables) and behaviour (methods) from one class to another.

  • Objective: Reusability, Avoid duplication.

Types of Inheritance

  • Single
  • Multi‑level
  • Hierarchical
  • Multiple (not supported by classes, but via interfaces)

Valid main() method signatures

public static void main(String[] args)   // standard
public static void main(String a[])       // valid
public static void main(String []a)       // valid
static public void main(String[] args)    // valid (order of modifiers allowed)
// invalid examples:
// void main(String args[]) public static
// public static void main(int a[])
📌 Note: The parameter name can be anything, but type must be String[].

📅 Day 16: Data Abstraction & Interface

Abstraction: Hiding implementation details and showing only functionality.

  • Achieved via abstract classes and interfaces.

Interface

  • Blueprint of a class.
  • Contains final & static variables.
  • Contains abstract methods (until Java 7); from Java 8 onwards allows default and static methods.
  • Abstract method: signature only, no body.
  • All methods are public by default.
  • Supports multiple inheritance.
  • Interface is defined using interface keyword.
  • A class implements an interface; an interface extends another interface.
  • We can create an object reference for an interface but cannot instantiate it.
interface I {
    int x = 10;  // public static final
    void m1();   // public abstract
}

class A implements I {
    public void m1() {
        System.out.println("Implemented");
    }
}

📊 Abstract Class vs Interface

Abstract ClassInterface
Use abstract keywordUse interface keyword
Subclasses extends abstract classSubclasses implements interface
Can have implemented methods + 0 or more abstract methodsJava 8+ can have default & static methods
We can extend only one abstract classWe can implement multiple interfaces
📌 Key point: Multiple inheritance is achieved only through interfaces in Java.

📅 Day 17: Wrapper Classes, Packages & Access Modifiers

Wrapper Classes

  • Convert primitives into objects and vice versa.
  • int → Integer, float → Float, double → Double, char → Character, boolean → Boolean.
  • Auto‑boxing: Primitive → Object automatically.
  • Unboxing: Object → Primitive automatically.
int x = 100;
Integer obj = x;   // auto-boxing
int y = obj;       // unboxing

// String to number
String s1 = "150";
int num = Integer.parseInt(s1);   // 150

// Number to String
String s2 = Integer.toString(123);

Packages

  • Built‑in packages (e.g., java.util, java.io)
  • User‑defined packages
  • Sub‑packages

Access Modifiers

  • public – accessible everywhere.
  • protected – accessible outside package only through inheritance.
  • default – accessible only within the same package.
  • private – accessible only within the class.

📅 Day 18: Type Casting (Upcasting / Downcasting)

Converting type of data.

  • Upcasting / Widening: Child to Parent reference.
  • Downcasting / Narrowing: Parent to Child reference (explicit cast).
Child c = new Child();
Parent p = new Child();   // upcasting (implicit)

// Downcasting (explicit)
Child c2 = (Child) p;

Type Casting Rules

Syntax: A b = (C) d;

  • Rule 1 (conversion valid?): Type of d and C must have a relationship (parent/child/same).
  • Rule 2 (assignment valid?): C must be same or child of A.
  • Rule 3 (runtime): Underlying object type of d must be same or child of C.
⚠️ Incorrect downcasting leads to ClassCastException.

📅 Day 19: Exception Handling

Exception: An event that causes program termination.

Types of Exceptions

  • Checked Exceptions: Identified by compiler (e.g., InterruptedException, FileNotFoundException, IOException). Must be handled or declared.
  • Unchecked Exceptions: Not identified by compiler (e.g., ArithmeticException, NullPointerException, ArrayIndexOutOfBoundsException).

try-catch-finally

try {
    // risky code
} catch (ExceptionType e) {
    // handle
} finally {
    // always executes – used to close resources
}

finally block behaviour

  • Exception occurs & catch handles → finally executes
  • Exception occurs & catch does NOT handle → finally executes
  • No exception → catch ignored, finally executes

Handling Unknown Exceptions

  • Multiple catch blocks (specific to general).
  • Single catch with Exception class.

throw vs throws

  • throw – used to explicitly throw an exception (inside method).
  • throws – used in method signature to declare that the method may throw an exception.
void readFile() throws IOException {
    throw new IOException("File not found");
}

📅 Day 20: Collections Framework

Collection: Represents a group of elements/objects/data as a single entity. Collection is an interface in java.util.

ArrayList (implements List)

  • Heterogeneous data – allowed
  • Insertion order preserved (index based)
  • Duplicate elements – allowed
  • Multiple null – allowed
import java.util.ArrayList;
ArrayList list = new ArrayList();
list.add(100);
list.add("Hello");
list.add(null);
list.add(null);
System.out.println(list);

HashSet (implements Set)

  • Heterogeneous data – allowed
  • Insertion order NOT preserved (no index)
  • Duplicate elements – NOT allowed
  • Multiple nulls – NOT allowed (only one null)
import java.util.HashSet;
HashSet set = new HashSet();
set.add(10);
set.add(20);
set.add(10);  // duplicate – ignored
set.add(null);
set.add(null); // second null ignored
System.out.println(set);

HashMap (implements Map)

  • Stores data as key‑value pairs.
  • Key must be unique, values can be duplicate.
  • Insertion order not preserved.
import java.util.HashMap;
HashMap map = new HashMap();
map.put(101, "John");
map.put(102, "Scott");
map.put(103, "Mary");
System.out.println(map.get(101)); // John

📝 Assignment: Employee & Group (Collections)

Problem:

  1. Create Employee class which contains id and name.
  2. Create Group class which contains groupID and employees list (ArrayList of Employee).
  3. Sort employee names (using Collections.sort() or custom comparator).

Solution Code

import java.util.*;

class Employee {
    int id;
    String name;
    Employee(int id, String name) {
        this.id = id;
        this.name = name;
    }
    public String toString() {
        return id + " : " + name;
    }
}

class Group {
    int groupID;
    ArrayList<Employee> employees;
    Group(int groupID) {
        this.groupID = groupID;
        employees = new ArrayList<>();
    }
    void addEmployee(Employee e) {
        employees.add(e);
    }
    void sortEmployeesByName() {
        // Using Comparator to sort by name
        Collections.sort(employees, (e1, e2) -> e1.name.compareTo(e2.name));
    }
    void display() {
        System.out.println("Group ID: " + groupID);
        for(Employee e : employees) {
            System.out.println(e);
        }
    }
}

// Main class to test
public class Main {
    public static void main(String[] args) {
        Group g = new Group(101);
        g.addEmployee(new Employee(3, "Mary"));
        g.addEmployee(new Employee(1, "John"));
        g.addEmployee(new Employee(2, "Alice"));
        
        System.out.println("Before sorting:");
        g.display();
        
        g.sortEmployeesByName();
        System.out.println("\nAfter sorting by name:");
        g.display();
    }
}
🎯 Expected Output:
Before sorting:
Group ID: 101
3 : Mary
1 : John
2 : Alice

After sorting by name:
Group ID: 101
2 : Alice
1 : John
3 : Mary
