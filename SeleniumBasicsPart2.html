1. Selenium WebDriver – Environment Setup & First Test

WebDriver is a Java interface (API) for automating browsers (Chrome, Firefox, Edge, etc.). Maven project is the recommended setup.

2. Locators – Overview & Basic Types

Locator = unique address of a web element. Selenium supports ID, Name, LinkText, PartialLinkText, ClassName, TagName.

LocatorMethodExampleReliability
IDBy.id("id")driver.findElement(By.id("search"))⭐ Highest
NameBy.name("name")driver.findElement(By.name("search"))⭐ High
LinkTextBy.linkText("exact")driver.findElement(By.linkText("Tablets"))✅ Good for <a>
PartialLinkTextBy.partialLinkText("partial")driver.findElement(By.partialLinkText("Tab"))⚠️ Avoid (ambiguous)
ClassNameBy.className("class")driver.findElements(By.className("list-inline-item"))🔹 Groups
TagNameBy.tagName("tag")driver.findElements(By.tagName("a"))🔸 Groups (links/images)
💡 Critical note: If class attribute contains spaces (multiple classes), using By.className("full class") will fail. Use CSS selector with first part or attribute selector.
Example: class="btn btn-primary" → use .btn or [class*='btn']. 
// ID example
WebElement logo = driver.findElement(By.id("logo"));
// Name example
driver.findElement(By.name("search")).sendKeys("MacBook");
// LinkText
driver.findElement(By.linkText("Tablets")).click();

3. findElement vs findElements – Detailed Comparison

ConditionfindElementfindElements
Single matchReturns WebElementList with 1 element
Multiple matchesReturns first elementList with all elements
No matchThrows NoSuchElementExceptionEmpty list (size 0)
 
// Safe check without exception
List<WebElement> elems = driver.findElements(By.id("maybe-missing"));
if(elems.size() > 0) {
    elems.get(0).click();
}

5. XPath – Absolute vs Relative

Absolute XPath (fragile): /html/body/div[2]/form/input — breaks on UI change.
Relative XPath (recommended): //input[@id='username'] — resilient, attribute-based.

// Basic relative XPath syntax
//tagname[@attribute='value']
//*[@id='search']   // any tag with id='search'

6. Advanced XPath – Functions, Operators, Dynamic Elements

6.1 text() – exact match

//a[text()='Desktops']

6.2 contains() – partial match

//input[contains(@placeholder,'Sea')]

6.3 starts-with()

//button[starts-with(@id,'temp_')]

6.4 AND / OR operators

//input[@name='search' and @type='text']
Dynamic ScenarioXPath Solution

Name prefix xyz001, xyz002//*[starts-with(@name,'xyz')]

7. XPath Axes – Multi‑Directional DOM Traversal

Navigate from a context node to parents, siblings, ancestors, descendants, following/preceding elements.

📐 XPath axes overview: Ancestor, Parent, Child, Following-sibling, Preceding-sibling from a given self node (Zomato link)

7.1 Axes Syntax & Examples

AxisSyntaxExample (context: //a[contains(text(),'Zomato')])
Parentparent::td//a[contains(text(),'Zomato')]/parent::td
Ancestorancestor::trgets whole table row
Followingfollowing::trall rows after current row
Precedingpreceding::trrows before current row
Following-siblingfollowing-sibling::trrows with same parent after

7.2 Complete Java Example – Rediff Gainers

WebDriver driver = new ChromeDriver();
driver.get("https://money.rediff.com/gainers/bse/daily/groupa");
String text = driver.findElement(By.xpath("//a[contains(text(),'Zomato')]/self::a")).getText();
System.out.println("Self text: " + text);
// ancestor row
WebElement row = driver.findElement(By.xpath("//a[contains(text(),'Zomato')]/ancestor::tr"));
List<WebElement> followingRows = driver.findElements(By.xpath("//a[contains(text(),'Zomato')]/ancestor::tr/following::tr"));
System.out.println("Following rows count: " + followingRows.size());
driver.quit();

9. Best Practices for Robust Locators

  • ✅ Prefer Relative XPath over absolute.
  • ✅ Use ID/name when available (fastest).
  • ✅ Avoid indexes [2] unless dynamic structure is fixed.
  • ✅ Validate locators in DevTools (Ctrl+F) before scripting.
  • ✅ For dynamic IDs, use contains() or starts-with().
  • ✅ Use findElements for optional elements to avoid exceptions.
  • ✅ Always close browser with driver.quit() in finally block.

11. Common Mistakes & Corrections

  • ❌ Using absolute XPath /html/body/div[3] → ✅ relative //div[@class='content']
  • ❌ Forgetting @ in XPath: //input[id='x'] → ✅ //input[@id='x']
  • ❌ Using text() on input fields → ✅ use @value or getAttribute("value")
  • ❌ Using class with spaces in CSS selector without dot → use .first-part
