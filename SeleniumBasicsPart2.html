Ultimate Guide to Selenium Locators

Basic Locators, CSS Selectors, XPath, and XPath Axes — complete reference for Automation Engineers, Testers & Developers with real-world examples, corrections, and diagrams.

1. Introduction to Locators

A locator is an address that uniquely identifies a web element. Without robust locators, tests become fragile. Selenium supports 8 strategies: ID, Name, LinkText, PartialLinkText, ClassName, TagName, CSS selectors, and XPath.

💡 Why master locators? Modern web apps are dynamic; many elements lack IDs; you need to handle iframes, shadow DOM, and changing attributes. Locators are the foundation of reliable automation.

Reference HTML snippet (used across examples):

<div id="wrapper">
  <div id="content">
    <form id="frmLogin" method="post">
      <div id="divUsername" class="textInputContainer">
        <input name="txtUsername" id="txtUsername" type="text">
        <span class="form-hint">Username</span>
      </div>
      ...
    </form>
  </div>
</div>

2. Basic Locators – ID, Name, LinkText, PartialLinkText, ClassName, TagName

2.1 ID Locator – Most Reliable

driver.findElement(By.id("txtUsername")).sendKeys("admin");

2.2 Name Locator

driver.findElement(By.name("txtUsername")).sendKeys("admin");

2.3 LinkText & PartialLinkText (only for <a>)

driver.findElement(By.linkText("Tablets")).click();   // preferred
// Avoid partialLinkText if possible

2.4 ClassName Locator – for groups

List<WebElement> items = driver.findElements(By.className("list-inline-item"));
⚠️ If class contains spaces (multiple classes), By.className fails. Use CSS selector instead: .firstClass.

2.5 TagName Locator – groups of same tag

driver.findElements(By.tagName("a"));   // all links

3. findElement vs findElements – Detailed Comparison

ConditionfindElementfindElements
Single matchReturns WebElementList with 1 element
Multiple matchesReturns first elementList of all elements
No matchThrows NoSuchElementExceptionEmpty list (size 0)
 
// safe check
List<WebElement> elems = driver.findElements(By.id("maybeMissing"));
if(elems.size() > 0) { elems.get(0).click(); }

4. CSS Selectors – Complete Guide

Faster than XPath, concise. Four main combinations:

PatternExampleSelector
Tag + IDinput#txtUsernameBy.cssSelector("#txtUsername")
Tag + Classinput.textInputContainerBy.cssSelector(".textInputContainer")
Tag + Attributeinput[name='txtUsername']By.cssSelector("[name='txtUsername']")
Tag + Class + Attrinput.textInputContainer[name='txtUsername']By.cssSelector(".textInputContainer[name='txtUsername']")
🧪 Validate in DevTools: Ctrl+F → paste CSS selector → shows matches count.
// Complete Java Example
driver.findElement(By.cssSelector("#small-searchterms")).sendKeys("T-shirts");
driver.findElement(By.cssSelector("input[placeholder='Search store']")).sendKeys("Laptop");

5. XPath – Absolute vs Relative

Absolute XPath (fragile): starts with /html/body/div[...] — breaks on any UI change.
Relative XPath (preferred): //input[@id='txtUsername'] — attribute-based, resilient.

Interview tip: Absolute XPath is rigid, relative XPath uses attributes and is stable.

6. Advanced XPath – Functions, Operators, Dynamic Elements

text() – exact inner text

//a[text()='Desktops']

contains() – partial match

//input[contains(@placeholder,'Sea')]

starts-with() – prefix matching

//div[starts-with(@id,'temp_')]

AND / OR operators

//input[@name='search' and @type='text']
//button[@id='startBtn' or @id='stopBtn']

Handling dynamic attributes (real scenarios)

Dynamic PatternXPath Solution
ID toggles startBtn/stopBtn//button[@id='startBtn' or @id='stopBtn']
ID always contains "st"//button[contains(@id,'st')]
Name prefix xyz001, xyz002//*[starts-with(@name,'xyz')]

Chained XPath (when element has no attributes)

//div[@id='logo']/a/img

Conditional XPath (comparisons)

//span[@class='price'][number(text()) > 100]

normalize-space() – ignore whitespace

//*[normalize-space()='Hello World']

7. XPath Axes – Multi‑Directional DOM Traversal

Axes let you navigate from a known element (context node) to related nodes: parents, siblings, ancestors, descendants, following/preceding.

🌲 DOM Tree – ancestors, descendants, siblings, parent-child relationships

Axes syntax and examples (context: //a[contains(text(),'Zomato')])

AxisSyntaxExample
selfself::a//a[contains(text(),'Zomato')]/self::a
parentparent::td//a[contains(text(),'Zomato')]/parent::td
ancestorancestor::trgets entire table row
following-siblingfollowing-sibling::trrows after current row (same parent)
preceding-siblingpreceding-sibling::trrows before current row
followingfollowing::trall following rows in document
📌 XPath Axes in action: navigating siblings and ancestors from a known element

Complete Java Example – XPathAxesDemo (Rediff Gainers)

WebDriver driver = new ChromeDriver();
driver.get("https://money.rediff.com/gainers/bse/daily/groupa");
String text = driver.findElement(By.xpath("//a[contains(text(),'Zomato')]/self::a")).getText();
System.out.println("Self : " + text);
// ancestor row
WebElement row = driver.findElement(By.xpath("//a[contains(text(),'Zomato')]/ancestor::tr"));
List<WebElement> followingRows = driver.findElements(By.xpath("//a[contains(text(),'Zomato')]/ancestor::tr/following::tr"));
System.out.println("Following rows: " + followingRows.size());
driver.quit();

8. SelectorHub – Auto‑Generation of Locators

Browser extension that generates relative CSS, XPath, absolute XPath, and validates uniqueness — handles shadow DOM and hidden elements.

✨ Productivity booster: Use SelectorHub to instantly build locators for Page Object Models, but always understand the underlying syntax.

9. Best Practices for Writing Robust Locators

  • ✅ Prefer Relative XPath over absolute
  • ✅ Use unique attributes: ID > name > contains() > axes
  • ✅ Avoid indexes ([2]) – they break with reordering
  • ✅ Validate locators using DevTools (Ctrl+F) before scripting
  • ✅ Use findElements for optional elements (no exception)
  • ✅ Use explicit waits (WebDriverWait) instead of Thread.sleep()
  • ✅ Always close browser with driver.quit() in finally block

10. Common Mistakes & Corrections

MistakeCorrection
Absolute XPath: /html/body/div[3]/formRelative: //form[@id='login']
Forgetting @: //input[id='x']//input[@id='x']
Using text() on input fieldsUse @value or getAttribute("value")
Full class with spaces in CSS: div.class1 class2Use .class1 or [class*='class1']
